## ğŸ“‹ DATABASE SCHEMA AT A GLANCE

BOOKS TABLE:
  - id (PK)
  - title, author, genre
  - isbn, description
  - average_rating, total_reviews
  - created_at, updated_at

REVIEWS TABLE:
  - id (PK)
  - book_id (FK to books)
  - book_title, author, genre
  - rating (1-5)
  - review_text
  - reviewer_name, reviewer_email
  - helpful_count
  - created_at, updated_at

USERS TABLE (ready for future authentication):
  - id, username, email, password_hash
  - full_name, bio, avatar_url
  - created_at, updated_at

USER_REVIEWS TABLE (links users to their reviews):
  - id, user_id (FK), review_id (FK)

RATINGS TABLE (separate rating tracking for analytics):
  - id, book_id (FK), user_id (FK), rating

BOOK_STATS VIEW (aggregated statistics):
  - Shows: id, title, author, total_reviews, average_rating, total_ratings
